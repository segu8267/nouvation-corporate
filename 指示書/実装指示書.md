# NOUVATION コーポレートサイト デザイン・実装仕様書

## 1. プロジェクト概要

- **サイト名:** NOUVATION (ノーベーション)
- **コンセプト:** **「Agri-Tech Core」**
  - 農業の現場感（土・重厚さ）× テクノロジー（近未来・光）の融合。
  - ターゲット（男性・農業従事者）に刺さる「プロツール」としての信頼感と、若手に響く「センス・先進性」の両立。
- **技術スタック:**
  - Framework: **Astro** (v4 系推奨)
  - Styling: **Tailwind CSS**
  - CMS: **MicroCMS** (ニュース・ブログ用)
  - Deploy: Vercel

## 2. デザインシステム (Design Rules)

### 2.1 カラーパレット

「黒・濃いグレー・深い緑」をベースに、「蛍光グリーン」で先進性を演出。

| 色名            | 変数名         | HEX       | 用途                                     |
| :-------------- | :------------- | :-------- | :--------------------------------------- |
| **Deep Soil**   | `bg-body`      | `#111111` | メイン背景色（黒）。                     |
| **Off White**   | `bg-sub`       | `#F4F4F5` | サブ背景色（白/グレー）。                |
| **Neon Lime**   | `accent`       | `#D4FF00` | アクセント。ボタン、強調線、あしらい。   |
| **Agri Green**  | `brand`        | `#1B5E20` | ブランドカラー。ロゴ、信頼性を示す箇所。 |
| **Matte Black** | `text-main`    | `#1F2937` | 白背景時の文字色。                       |
| **Smoke White** | `text-inverse` | `#E5E7EB` | 黒背景時の文字色。                       |

### 2.2 タイポグラフィ

画面幅いっぱいに使う英語フォントと、可読性の高い日本語フォント。

- **欧文 (Heading/Display):** **'Syne'** or **'Unbounded'** (Google Fonts)
  - 横幅が広い(Wide)、幾何学的、近未来感。
- **和文 (Body):** **'Noto Sans JP'** (Google Fonts)
  - Weight: 400(Regular), 700(Bold)。

### 2.3 質感・あしらい

- **ノイズ加工:** `body` 全体にうっすらと粒子状のノイズテクスチャを配置（CSS `::before` 等で実装）し、密度感を出す。
- **角丸 (Radius):** `rounded-2xl` (16px~24px) を基本とする。
- **グリッド:** 装飾として細い線（border）やグリッド背景を薄く配置し、設計図のようなテクニカルさを演出。

## 3. レイアウト・スペーシング規定 (Tailwind ベース)

### 3.1 セクション構成

- **セクション間余白:** `py-20` (80px) 〜 `py-24` (96px)
  - ゆったりとした余白を取り、高級感を出す。
- **コンテンツ最大幅:** `max-w-7xl` (1280px)
  - 文章の読みやすさを考慮し、テキストエリアは `max-w-3xl` 等で絞る。

### 3.2 グリッドシステム

- **基本:** 12 カラムグリッドを意識するが、実装は CSS Grid/Flexbox を使用。
- **ガター (Gap):**
  - PC: `gap-8` (32px) 〜 `gap-12` (48px)
  - SP: `gap-6` (24px)

## 4. コンポーネントデザイン

### 4.1 ボタン (Button)

- **Primary (CTA):**
  - 背景: `brand` (#1B5E20) または `accent` (#D4FF00) ※黒背景時
  - 文字: 白 または 黒（視認性重視）
  - 形状: `rounded-full` (丸型)
  - ホバー: 色を少し明るく/暗く + `translate-y-1` (少し浮く) + 影 (`shadow-lg`)
- **Secondary (Ghost):**
  - 背景: 透明
  - 枠線: 1px solid
  - ホバー: 背景塗りつぶし

### 4.2 カード (Card)

- **スタイル:**
  - 背景: 白 (`#FFFFFF`) または わずかなグレー
  - 枠線: `border border-gray-200`
  - 角丸: `rounded-2xl`
  - 影: `shadow-lg` (ホバー時に `shadow-xl` へ拡大)
  - 動き: ホバー時に画像が少し拡大 (`scale-105`) する等のマイクロインタラクション。

### 4.3 フォーム (Form)

- **Input/Textarea:**
  - 背景: 薄いグレー (`bg-gray-50`)
  - 枠線: 下線のみ、または薄いボーダー。フォーカス時に `accent` カラーで発光。
  - 角丸: `rounded-lg`

### 4.4 リンク (Text Link)

- **通常:** 下線なし、または目立たない色。
- **ホバー:** `accent` または `brand` カラーに変化 + 下線アニメーション（左から右へ伸びる）。

## 5. ギミック・アニメーション要件

1. **スマートヘッダー (Smart Header):**
   - 初期: 背景透明・白文字。
   - スクロールダウン: ヘッダー非表示。
   - スクロールアップ: ヘッダー表示（背景: 黒のすりガラス `backdrop-blur`）。
2. **無限マーキー (Infinite Marquee):**
   - フッター最下部に画面幅をはみ出す巨大な「NOUVATION」の文字。
   - CSS animation で左から右へゆっくり無限ループ。
3. **スクロール出現 (Fade Up):**
   - `IntersectionObserver` を使用。
   - 要素が画面内に入ったら `opacity: 0 -> 1`, `translateY: 20px -> 0`。

## 6. コンテンツ・セクション構成

### ① FV (First View)

- **背景:** 動画（MP4/WebM）をフルスクリーン配置。暗めのオーバーレイを重ねる。
- **コピー:** 「農業界に、情報で革新を。」（極太フォント・白抜き）。
- **UI:** 画面下部に「SCROLL」を促す矢印アニメーション。

### ② NEWS

- **MicroCMS 連携前提:** 表示用データは一旦 JSON 配列のダミーで実装。
- **デザイン:** 日付・カテゴリ・タイトルのみのミニマルなリスト。

### ③ Social Issues

- **背景:** 黒 (`#111111`)。
- **演出:** 重厚なテキストで課題（情報格差・孤立）を訴求。

### ④ Mission

- **レイアウト:** テキスト(左) + イメージ画像(右)。
- **画像:** 農業 × つながりを感じさせる抽象的でエモーショナルな写真。

### ⑤ Services (4 つの柱)

- **Service 01 (Market Graph):** グラフ UI のモックアップ（静止画または CSS 描画）。
- **Service 02 (Instagram):** スマホ投稿風の UI。
- **Service 03 (AI Support):** チャット UI。「COMING SOON」バッジをネオンカラーで。
- **Service 04 (Web Production):** PC/スマホのデバイスモックアップ。

### ⑥ Founder's Story

- **画像:** 代表のシルエットや後ろ姿など、雰囲気のある写真。名前は「Founder / CEO」のみ。

### ⑦ Voice

- **カード:** 横並びグリッド。ユーザーアイコンはプレースホルダー。

### ⑧ Footer

- **背景:** 黒。
- **要素:** サイトマップ、特商法・プライバシーポリシーへのリンク、LINE 登録ボタン、Marquee アニメーション。

## 7. コーディング・命名規則

- **HTML 構造:** セマンティックなマークアップ (`header`, `main`, `section`, `footer`, `article`) を徹底する。
- **CSS 設計 (Tailwind + BEM):**
  - 基本は **Tailwind CSS のユーティリティクラス** でスタイリングする。
  - `.astro` ファイル内の `<style>` タグで独自の CSS を書く場合（アニメーションや複雑な疑似要素など）は、**BEM 記法** を採用する。
  - 例:

    ```css
    /* 悪い例 */
    .marquee-text {
      ...;
    }

    /* 良い例 (BEM) */
    .footer__marquee {
      ...;
    }
    .footer__marquee-text {
      ...;
    }
    ```
- **レスポンシブ:** モバイルファーストで記述（`class="..."` が SP、`md:...` が PC）。
- **画像:** `loading="lazy"`、`width/height`指定を必須とし、CLS 対策を行う。
